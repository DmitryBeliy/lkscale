## 🎯 Сценарии тестирования

### 1. Авторизация и онбординг

#### TC-001: Успешный вход
```gherkin
Feature: Авторизация

Scenario: Успешный вход существующего пользователя
  Given пользователь на экране входа
  When вводит email "manager@store.demo"
  And вводит пароль "Demo123!"
  And нажимает "Войти"
  Then открывается Dashboard
  And отображается имя пользователя
```

#### TC-002: Неверный пароль
```gherkin
Scenario: Попытка входа с неверным паролем
  Given пользователь на экране входа
  When вводит email "manager@store.demo"
  And вводит пароль "WrongPass123"
  And нажимает "Войти"
  Then отображается ошибка "Неверный email или пароль"
  And остается на экране входа
```

#### TC-003: Биометрический вход
```gherkin
Scenario: Вход по отпечатку/лицу
  Given пользователь включил биометрию в настройках
  And ранее входил в приложение
  When открывает приложение
  And нажимает "Войти по отпечатку"
  Then происходит биометрическая аутентификация
  And при успехе открывается Dashboard
```

### 2. Управление товарами

#### TC-101: Создание товара
```gherkin
Feature: Управление товарами

Scenario: Создание нового товара
  Given пользователь на экране "Склад"
  When нажимает "+ Добавить товар"
  And заполняет поля:
    | Поле | Значение |
    | Название | Тестовый товар |
    | SKU | TEST-001 |
    | Цена продажи | 5000 |
    | Себестоимость | 3000 |
    | Остаток | 100 |
  And выбирает категорию "Электроника"
  And нажимает "Сохранить"
  Then товар сохраняется
  And появляется в списке товаров
  And отображается уведомление "Товар создан"
```

#### TC-102: Редактирование товара
```gherkin
Scenario: Изменение цены товара
  Given пользователь открыл карточку товара "iPhone 15 Pro"
  When нажимает "Редактировать"
  And меняет цену на "130000"
  And нажимает "Сохранить"
  Then цена обновляется
  And история цен содержит запись об изменении
```

#### TC-103: Удаление товара
```gherkin
Scenario: Удаление товара из каталога
  Given пользователь открыл карточку товара
  When нажимает "Удалить"
  And подтверждает действие
  Then товар помечается как неактивный
  And исчезает из списка активных
```

#### TC-104: Сканирование штрих-кода
```gherkin
Scenario: Добавление товара по штрих-коду
  Given пользователь на экране добавления товара
  When нажимает иконку сканера
  And сканирует штрих-код "4601234567890"
  Then поле штрих-кода заполняется автоматически
```

#### TC-202: Перемещение между складами
```gherkin
Scenario: Перемещение товара
  Given есть 2 активных склада

  And выбирает склад-отправитель "Склад 1"

  And добавляет товар "MacBook Air M3" количество 2

  And на складе-отправителе резервируется товар
```

## 🔍 Особые сценарии

### Нагрузочное тестирование

```gherkin
Scenario: Работа с большим каталогом
  Given в системе 10,000+ товаров
  When пользователь открывает каталог
  Then список загружается за < 2 сек
  And прокрутка работает плавно (60 FPS)
  And поиск отрабатывает за < 500ms
```

### Тестирование безопасности

```gherkin
Scenario: Проверка доступа
  Given кассир авторизован в системе
  When пытается открыть CFO Dashboard
  Then отображается ошибка доступа
  And действие логируется
```

### Тестирование граничных случаев

| Сценарий | Ожидаемый результат |
|----------|---------------------|
| Товар с ценой 0 | Ошибка валидации |
| Заказ без товаров | Кнопка "Создать" disabled |
| Отрицательный остаток | Предупреждение, блокировка продажи |
| Очень длинное название (>100 симв) | Обрезка + многоточие |
| Спецсимволы в полях | Экранирование, XSS защита |

## 🤖 Автоматизация

### Unit Tests

```bash
# Запуск всех тестов
npm test

# Запуск с coverage
npm test -- --coverage

# Запуск конкретного файла
npm test -- store/dataStore.test.ts
```

### E2E Tests (Detox)

```bash
# iOS
npm run e2e:ios

# Android
npm run e2e:android
```

### CI/CD Pipeline

```yaml
name: Test Suite
on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install dependencies
        run: npm ci
      - name: Run linter
        run: npm run lint
      - name: Type check
        run: npx tsc --noEmit
      - name: Run tests
        run: npm test -- --coverage
```

## 📞 Обратная связь

При обнаружении багов создавайте issue с:

1. **Environment**: OS, версия приложения

3. **Expected result**: Ожидаемое поведение
4. **Actual result**: Фактическое поведение
5. **Screenshots/Videos**: Визуальное подтверждение
